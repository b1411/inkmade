## 0. ЗАФИКСИРОВАНО — НЕ МЕНЯТЬ (дизайн и шрифты)

Заказчику нравятся дизайн-направление и шрифты. **Эти решения заблокированы.** Менять только по явному точечному запросу («мелкие правки»), не трогать по своей инициативе при любой другой работе.

**Якоря (не переписывать без запроса):**
- Дизайн-токены: [app/assets/css/main.css](app/assets/css/main.css) — палитра, тени, типошкала, отступы, аним-токены, grain, `::selection`.
- Шрифты: **Unbounded** (заголовки, кириллица), **Manrope** (текст/UI), **Space Mono** (лейблы), **Permanent Marker** (только логотип). Конфиг — [nuxt.config.ts](nuxt.config.ts) `fonts`.
- Базовые компоненты: `UiAppButton`, `UiAppCard`, `UiSkeleton`, `UiEmptyState`, `UiReveal`, `UiSectionLabel`, `UiCustomCursor`, `LayoutAppHeader`/`LayoutAppFooter`.
- Движение: плагины Lenis/GSAP, `v-magnetic`, `useReducedMotion`, page transitions.
- Лендинг-секции и их копирайт (ТЗ §5).

**Правило:** новый функционал собираем ИЗ этих компонентов и токенов. Цвета/радиусы/тени/шрифты берём только из переменных `main.css`. Любая новая страница обязана выглядеть как часть этой системы.

## 6. БЕЗОПАСНОСТЬ — HARDENING (конкретно, по находкам)

### CRITICAL
1. **RPC EXECUTE — эскалация прав (Supabase advisor, 13 функций).** Любой залогиненный юзер может вызвать через `/rest/v1/rpc/...`: `admin_stats`, `admin_finance_stats/series`, `admin_margin_breakdown`, `admin_list_users`, `set_royalty_rate`, `mark_payout_paid`, `refund_order`, `adjust_stock`, `studio_get_order`, `studio_list_queue`, `request_payout`, `claim_designer_invite`.
   → Миграция: `REVOKE EXECUTE ON FUNCTION ... FROM authenticated, anon;` (оставить service_role), и/или гарантировать внутренний guard `private.is_admin()/is_staff()` в каждой. Перепроверить каждую функцию вручную. Док: database-linter lint=0029.
2. **Rate limiting — расширить.** Сейчас `server/middleware/rate-limit.ts` покрывает payment/orders. Добавить лимиты на: `designs/shared/[token]` (перебор токенов), `promo/validate` (перебор кодов), `designs/import`, и проксировать/ограничить попытки логина (Supabase Auth + капча).
3. **Загрузка файлов — серверная валидация.** Принты грузятся клиентом прямо в Storage; MIME `image/*`+svg+pdf слишком широко. → Проверка magic-bytes (`file-type`), **санитайз SVG** (strip `<script>`/on*; svgo/DOMPurify) или запрет SVG, сузить MIME до png/jpeg/webp(+контролируемый svg/pdf), серверный лимит размера.
4. **Zod-валидация всех `readBody`.** `orders/create` (структура `shipping_addr`: email/phone/город/адрес), `designs/import` (`spec.placements`, `previewUrl` как URL), `promo` (длина/символы), webhook (`providerTxn`).

### HIGH (перф-БД и доступ — Supabase advisor, 171 линт)
5. `auth_rls_initplan` (20): RLS-политики вызывают `auth.uid()` per-row → заменить на `(select auth.uid())`. Миграция.
6. `multiple_permissive_policies` (115): консолидировать дублирующие permissive-политики (по таблице/роли/действию) — меньше оверхед на запрос.
7. `unindexed_foreign_keys` (23): добавить индексы на FK (джойны/каскады).
8. `unused_index` (12): удалить неиспользуемые индексы.

### MEDIUM
9. Supabase Auth: включить подтверждение email, политику паролей, **leaked-password protection (HIBP)**, капчу на login/register.
10. CSP: перейти на `nonce` для script когда Nuxt позволит (сейчас `unsafe-inline` вынужденно). Доп. заголовки при необходимости.
11. Логирование подозрительной активности (низкие суммы, много промокодов, webhook-failures). Процесс ротации `PAYMENT_WEBHOOK_SECRET`.
12. Не утекать `error.message` БД в ответы (designs/moderation и пр.).
