# B2B-магазины на субдоменах («INKMADE для команд») — исследование и план

> Статус: план утверждается. Домен inkmade.kz ещё не куплен — архитектура строится так,
> чтобы работать по пути (`/s/uib`) сегодня и переключиться на субдомен (`uib.inkmade.kz`)
> одним флагом, когда домен появится.

## 1. Исследование

### 1.1 Всемайки (vsemayki.ru) — как у них

Экосистема из 4 продуктов, каждый на своей посадочной:

| Продукт | Где | Механика |
|---|---|---|
| Партнёрка вебмастерам | webmasters.vsemayki.ru | свой магазин на платформе, **25% с оплаченного заказа**; тариф «Лайт» бесплатно («магазин за 10 минут»), «Про» — REST API |
| Блогерам | bloggers.vsemayki.ru | «Мерч без вложений»: форма 4 поля (имя, телефон, email, ссылка на соцсеть) → магазин за минуты → менеджер на связи; штатные дизайнеры бесплатно доводят макеты |
| Дизайнерам | designers.vsemayki.ru | принты в общий каталог за роялти ~3% |
| Опт / B2B | vse.ru | через менеджера, от 1 шт., образец за 48 ч |

- **URL-модель:** поддомен на **отдельном** «магазинном» домене — `название.vsemaykishop.ru`; свой домен паркуется через NS. Корзина/чекаут платформенные.
- **Производство:** print-on-demand, 48 ч, «не храним запасы», колл-центр и доставка на стороне платформы.
- **Подача на главной: слабая** — только пункт «Опт» в меню и блок «Сотрудничество» в футере. Отдельной секции «открой магазин» на главной НЕТ → **наша возможность дифференцироваться заметной секцией**.

### 1.2 Мировые практики (Spreadshop, Fourthwall, Spring, Printify, corporate-swag)

| Платформа | URL | Экономика | Ключевое |
|---|---|---|---|
| Spreadshop | `name.myspreadshop.com` + свой домен | base price + **наценка владельца = его прибыль**; payout от $10/мес; volume-тиры снижают base | JS-embed магазина в свой сайт, WP-плагин |
| Fourthwall | `name.shop.fourthwall.com` + свой домен | без monthly fee, владелец ставит цену, «100% профита» сверх cost | memberships, YouTube shelf, TikTok Shop, «мы делаем саппорт покупателей <12h» |
| Spring | поддомен | base cost + retail владельца (~$8–10 с футболки) | встроенный простой дизайнер |
| Printify Pop-Up | `name.printify.me` | Stripe-автовыплаты за вычетом base | секционный no-code билдер с live-preview, персонализация покупателем, AI-генератор |
| Corporate swag (Axomo, Printfection…) | закрытые сторы | бюджеты/кредиты по ролям | login-gate, approvals, сегментация витрин, дашборд redemption |

**10 паттернов, которые повторяются у всех (берём):**
1. Мгновенный бесплатный поддомен; свой домен — как апгрейд.
2. Base price прозрачен, **наценку ставит владелец** → нарратив «no fees, маржа — твоя».
3. **3-шаговый лендинг**: «создай → мы печатаем и шлём → делись ссылкой».
4. No-code билдер витрины с live-preview (лого, цвета, баннер, коллекции).
5. Магазин идёт к аудитории: embed-виджет, QR, соцсети.
6. Payout-центр: автоначисления, минимальный порог, volume-скидки.
7. Платформа берёт саппорт покупателей и гарантии (reprint/refund) — снимает страх владельца.
8. Realtime-аналитика продаж.
9. Corporate-режим = закрытый стор (код доступа) + бюджеты + approvals — отдельный продукт на той же инфре.
10. Слои сверху: персонализация покупателем, memberships.

## 2. Ключевое преимущество INKMADE: инфраструктура уже наполовину есть

Механика дизайнерского маркетплейса (сейчас за флагом `designerMarketplace`) — это 80% механики B2B-магазинов:

| Нужно магазину | Уже есть |
|---|---|
| Библиотека принтов владельца + модерация | `print_library` (owner_id, moderation_status, guard-триггер) |
| Начисление доли с каждой продажи | `apply_paid`: атрибуция `print_id`/`print_owner_id` → `royalty_earnings` → `designer_balances` (anti-fraud «принт реально использован» уже встроен) |
| Выплаты | `payouts` + RPC `request_payout` + админ `mark_payout_paid` |
| Профиль/витрина владельца | `designer_profiles` + публичная RPC-проекция |
| Инвайт владельца по email | `designer_invitations` + admin invite (Auth Admin API) |
| Конструктор для composability дизайнов | customizer (Konva) + `designs` (spec, preview, print_file) |
| Производство/статусы/цех | весь orders-пайплайн + /studio |
| Фиче-гейтинг | `shared/config/features.ts` + feature-flags middleware |

**Следствие:** money-механика v1 (= % с продаж магазину) запускается почти без нового кода — при активации магазина создаём владельцу internal-запись `designer_profiles` с `royalty_pct = доля магазина`, и `apply_paid` уже всё начисляет.

## 3. Архитектурные решения

### 3.1 URL и мульти-тенантность (домен не куплен — не блокер)
- **Tenant-resolver** — одна серверная утилита, определяющая магазин: (а) по субдомену `Host`-заголовка, (б) по пути `/s/[slug]`. Результат кладётся в `useState('shop')` на SSR.
- **Сейчас:** path-режим `/s/uib` — работает на текущем vercel.app.
- **Когда куплен домен:** Vercel wildcard `*.inkmade.kz` + включаем host-режим. Страницы/компоненты не меняются.
- Auth-cookie: при переходе на субдомены выставить cookie-domain `.inkmade.kz` (единый логин магазин↔платформа).
- Зарезервированные slug'и: `www, api, admin, studio, app, mail, shop, cdn…`.
- (Отметить: Всемайки держит магазины на отдельном домене `vsemaykishop.ru` — вариант `*.inkmade.shop` на будущее, если захотим изолировать SEO/cookie; не обязательный.)

### 3.2 Модель данных (новая миграция)
```
shop_applications  — заявка с лендинга: org_name, contact_name, phone, email,
                     desired_slug, comment, status(pending/approved/rejected),
                     created_at, resolved_by/at.  RLS: anon insert, admin read/update.

shops              — id, slug UNIQUE (subdomain-safe, reserved-list check),
                     name, status(active/suspended), owner_id → auth.users,
                     logo_url, theme jsonb (primary/bg/accent цвета),
                     hero jsonb (title, subtitle, banner_url),
                     contacts jsonb (instagram, phone, whatsapp),
                     revenue_share_pct numeric, access_code text NULL (закрытый стор),
                     is_public bool, created_at.
                     RLS: anon read active+is_public; owner update своих
                     НЕ-критичных полей; slug/status/revenue_share — только admin
                     (guard-триггер, паттерн guard_orders_update).

print_library      — + shop_id uuid NULL: принты магазина не попадают в общий
                     каталог, модерация той же админ-очередью.

shop_items         — витрина: id, shop_id, design_id → designs (композиция из
                     конструктора: товар+вариант+принт+превью), title, description,
                     sort, is_active, markup numeric NULL (v2).
                     RLS: anon read active (join shops.active), owner CRUD.
```
Атрибуция заказа магазину — **уже есть** через `order_items.print_id/print_owner_id`; для отчётности добавим `order_items.shop_id` (денормализация, заполняет сервер).

### 3.3 Роли — новой роли НЕ нужно
Владелец магазина = обычный `customer` + `shops.owner_id` (как `designer_profiles`). Middleware `shop-owner` проверяет наличие активного магазина. Один человек может быть и покупателем, и владельцем. Admin/operator работают как раньше.

### 3.4 Экономика
- **v1 (запуск):** фиксированная доля магазина с продажи (например 15–25%, поле `revenue_share_pct`, задаёт админ при активации) — через существующий royalty-пайплайн. Просто и «как у Всемайки» (у них 25%).
- **v2 (сила, как у мира):** владелец сам ставит **наценку на каждый товар** (`shop_items.markup`) — «розничная цена = наша базовая + твоя наценка, наценка целиком твоя». Требует: капчер markup в `order_items` + ветка в `apply_paid`. Это главный мировой паттерн — планируем сразу после v1.

## 4. План по фазам

### B0 — Решения и каркас (полдня)
Флаг `FEATURES.b2bShops=false` + маршруты в FEATURE_ROUTES; нейминг секции (рабочее: «Магазин мерча для вашей команды»); список reserved slugs; фиксация экономики v1.

### B1 — Лендинг + заявки (1–2 дня) ← **запускается ПЕРВЫМ, до готовности функционала**
- Секция на главной (после HowItWorks): заголовок, 3 шага («Оставьте заявку → Открываем магазин вашей команды → Добавляете дизайны — мы печатаем и доставляем каждый заказ»), value-props (без вложений и склада · печать от 1 шт · брендированная витрина · доля с каждой продажи), CTA.
- Посадочная `/business`: расширенная подача + FAQ + **форма заявки** (организация, контакт, телефон, email, желаемый адрес `___.inkmade.kz`, комментарий) → `shop_applications`.
- Админ-очередь заявок (`/admin/shops`): просмотр, approve/reject; approve = создать shop + инвайт владельца по email (существующая инвайт-инфра).
- **Ценность сразу: собираем B2B-лиды ещё до постройки магазинов** (и до покупки домена).

### B2 — Ядро тенантов + публичная витрина MVP (2–4 дня)
- Миграция (`shops`, `shop_applications`, `shop_items`, `print_library.shop_id`, RLS, guard-триггеры).
- Tenant-resolver (host+path) + layout `shop.vue`: тема через CSS-переменные из `theme`, лого, hero-баннер, футер «Работает на INKMADE».
- Витрина: сетка shop_items → карточка товара (превью композиции, размеры/цвета из вариантов, цена) → корзина/чекаут **общие** (заказ падает в тот же цех).
- `order_items.shop_id` атрибуция. Закрытый режим по `access_code` (corporate-паттерн).

### B3 — Кабинет владельца (3–5 дней)
`/shop-admin` (за флагом): Дашборд (продажи, начисления — reuse designer-finance UI) · Витрина (CRUD shop_items: выбор товара из нашего каталога → конструктор с принтами магазина → сохранение композиции) · Принты (upload → модерация админом) · Брендинг (лого, цвета, hero — live-preview) · Финансы (баланс, запрос выплаты — существующие RPC) · Настройки (контакты, закрытый режим, QR-код ссылки).

### B4 — Money v1 (1 день)
Provisioning при активации: internal `designer_profiles` для владельца с `royalty_pct = revenue_share_pct` → `apply_paid` начисляет автоматически. Финансы магазина в админке уже видны через существующие отчёты.

### B5 — Power-фичи (итеративно, по мировым паттернам)
Наценка владельца v2 (главное) · embed-виджет витрины (JS, как Spreadshop) · коллекции/сегменты витрины · realtime-аналитика магазина · персонализация покупателем (имя/номер на спине) · bulk-заказ для HR (корзина размеров одним заказом) · volume-скидки base price · авто-тема из логотипа.

### B6 — Субдомены (когда куплен домен, ~1 день)
Vercel wildcard `*.inkmade.kz` → включить host-режим резолвера → cookie-domain `.inkmade.kz` → OG/sitemap per shop → (позже) парковка собственных доменов партнёров, как у Всемайки/Spreadshop.

## 5. Риски и зависимости
- **Платёж = mock (блокер #1)** — реальные деньги магазинов требуют реального провайдера; лендинг+заявки (B1) и вся постройка (B2–B4) от этого не зависят.
- Модерация контента: копирайт-согласия и модерация принтов уже есть; для B2B добавить причину отклонения в UI владельца.
- `designerMarketplace` остаётся выключенным — B2B-магазины НЕ включают его: shop-принты гейтятся `shop_id`, витрины дизайнеров не открываются.
- Суспенд магазина (нарушения) — статус + 404 витрины, деньги замораживаются в балансе.
- i18n ru+kk с первого дня, как везде в проекте.
