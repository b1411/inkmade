# INKMADE — Дизайн-ТЗ для разработчика

Полное техническое задание по визуалу, UX, анимациям и копирайтингу. Цель: превратить функциональный сайт в продукт, который выглядит дороже, чем стоил, и вызывает доверие у аудитории 18–35 за 2 секунды. Весь копирайтинг в документе — финальный, готовый к вставке (не плейсхолдеры).

Стек (зафиксирован): Nuxt 4 + Vue 3 + Nuxt UI v4 + Tailwind v4 + Konva. Цвета #7A1F28 / #111111 / #EFE0C1.

---

## 1. Философия и направление дизайна

### 1.1 Направление: «премиальный стрит-минимализм»
Чистая дорогая структура (воздух, крупная типографика, большие изображения — язык Apple/Gymshark) + characterful-акценты бренда (бордо, фактура, дисплейный шрифт дозированно). Спокойный дорогой каркас, в который врезаются яркие бренд-моменты.

Почему так: аудитория 18–35 считывает качество бренда по визуалу за 2 секунды и переносит это суждение на качество физического товара. Плохой/шаблонный сайт = «наверное, и печать так себе». Сайт должен выглядеть как у established-бренда, иначе не продаст.

### 1.2 Три закона (не нарушать)
1. **Воздух и крупность.** Много негативного пространства, крупные заголовки, большие фото. Тесный UI = дешёвый UI. Дорогие бренды не боятся пустоты.
2. **Один акцент на экран.** Бордо — в одной ключевой точке экрана (главный CTA, активный элемент). Остальное — нейтральная база. Размазанный по всему экрану акцент убивает премиальность.
3. **Движение = качество.** 80% «дорогого» ощущения создают микроанимации: появление при скролле, реакция на hover, плавные переходы, инерционный скролл. Без движения даже идеальная вёрстка ощущается статичной и дешёвой.

### 1.3 Что НЕ делать
- Граффити-фон на всю страницу (убивает читаемость, отпугивает часть аудитории). Характер — в акцентах, не в хаосе.
- Дефолтный вид Nuxt UI без кастомизации (похоже на сотни сайтов).
- Анимации ради анимаций (раздражают, тормозят). Сдержанно и быстро.
- Стоковые 3D-рендеры вещей. Только реальные фото на людях.
- Дисплейный граффити-шрифт в body или мелком тексте (читается подростково/дёшево).

## 2. Дизайн-токены (расширенные)

### 2.1 Цвета (полная палитра)
```css
:root {
  /* Бренд */
  --ink-burgundy: #7A1F28;      --ink-burgundy-dark: #5E1820;   --ink-burgundy-light: #9A3540;
  --ink-black: #111111;          --ink-black-soft: #1E1E1E;
  --ink-cream: #EFE0C1;          --ink-cream-dark: #D7C9A6;
  /* Нейтральные (тёплые, не холодные) */
  --ink-white: #FBF8F2;          /* тёплый белый, фон светлых секций */
  --ink-gray-50:  #F4F1EA;       --ink-gray-200: #E5E0D8;
  --ink-gray-400: #9A9A9A;       --ink-gray-600: #555555;        --ink-gray-900: #1A1A1A;
  /* Семантика */
  --ink-success: #3E7C5A;        --ink-warning: #C8922A;         --ink-error: #B23A3A;
  /* Заливки/эффекты */
  --ink-overlay: rgba(17,17,17,0.6);     /* затемнение на фото */
  --ink-glass: rgba(251,248,242,0.7);    /* стекло на светлом */
}
```
Правило применения: тёмные секции — фон `--ink-black` или `--ink-burgundy`, текст `--ink-cream`. Светлые — фон `--ink-white`/`--ink-gray-50`, текст `--ink-black`. Бордо как заливка — только CTA и акцент. Холст кастомайзера — всегда нейтральный (`--ink-gray-50`).

### 2.2 Типографика
```css
:root {
  --font-display: 'Clash Display', 'Bebas Neue', sans-serif;  /* заголовки: характерный гротеск */
  --font-body: 'Manrope', sans-serif;                          /* текст, UI, кириллица */
  --font-mono: 'Space Mono', monospace;                        /* лейблы, цифры, теги */
}
```
**Решение по шрифту:** заменить граффити-маркер на характерный современный гротеск (Clash Display или General Sans / Cabinet Grotesk — все бесплатны на Fontshare, имеют кириллицу или латинскую выразительность). Граффити-леттеринг оставить ТОЛЬКО в логотипе как акцент бренда. Это сдвигает ощущение от «подростковая печать» к «современный бренд», не теряя характера.

Шкала (clamp для адаптива):
```css
--text-hero:    clamp(2.75rem, 7vw, 6rem);     /* H1 hero, display, line-height 0.95 */
--text-h1:      clamp(2rem, 4vw, 3.5rem);       /* display */
--text-h2:      clamp(1.5rem, 3vw, 2.5rem);     /* display */
--text-h3:      1.5rem;                          /* body 700 */
--text-lead:    1.25rem;                         /* подзаголовки, body 400 */
--text-body:    1.0625rem;                       /* основной, line-height 1.6 */
--text-sm:      0.875rem;
--text-label:   0.75rem;   /* mono, uppercase, letter-spacing 0.18em */
```
Заголовки display — letter-spacing -0.02em (плотно, дорого). Лейблы mono — uppercase, letter-spacing 0.18em (приём из сабтекста лого).

### 2.3 Пространство, радиусы, тени
```css
:root {
  /* Шкала отступов (кратна 4) */
  --space-2: 8px; --space-3: 12px; --space-4: 16px; --space-6: 24px;
  --space-8: 32px; --space-12: 48px; --space-16: 64px; --space-24: 96px; --space-32: 128px;
  /* Секции лендинга — крупные вертикальные отступы */
  --section-pad: clamp(64px, 10vw, 128px);
  /* Радиусы */
  --radius-sm: 8px; --radius-md: 14px; --radius-lg: 24px; --radius-xl: 32px; --radius-full: 9999px;
  /* Тени — мягкие, многослойные (дорогие), не резкие */
  --shadow-sm: 0 1px 2px rgba(17,17,17,0.04), 0 2px 6px rgba(17,17,17,0.06);
  --shadow-md: 0 4px 12px rgba(17,17,17,0.08), 0 12px 28px rgba(17,17,17,0.10);
  --shadow-lg: 0 8px 24px rgba(17,17,17,0.10), 0 24px 60px rgba(17,17,17,0.14);
  --shadow-burgundy: 0 8px 24px rgba(122,31,40,0.28);  /* свечение под бордо-CTA */
  --container: 1280px;
}
```

### 2.4 Анимационные токены
```css
:root {
  --ease-out: cubic-bezier(0.22, 1, 0.36, 1);       /* основной, мягкое торможение */
  --ease-in-out: cubic-bezier(0.65, 0, 0.35, 1);
  --ease-spring: cubic-bezier(0.34, 1.56, 0.64, 1);  /* лёгкий overshoot для «живых» элементов */
  --dur-fast: 180ms; --dur-base: 320ms; --dur-slow: 520ms; --dur-reveal: 700ms;
}
```
Правило: интерфейсные реакции (hover, клик) — fast/base. Появление секций — reveal. Easing по умолчанию — `--ease-out`. Overshoot (`--ease-spring`) — только для игривых акцентов (кнопка, бейдж), не для крупных блоков.

## 3. Библиотеки (полный список с назначением)

### 3.1 Анимация — ядро «дорогого» ощущения
| Библиотека | Назначение | Где применять | Приоритет |
|---|---|---|---|
| **Lenis** | инерционный плавный скролл | вся страница | СТАВИТЬ ПЕРВЫМ — максимальный эффект за минимум усилий |
| **@vueuse/motion** | появление при скролле, hover, fade/slide | секции лендинга, карточки | высокий, Vue-нативный |
| **GSAP** + ScrollTrigger | сложные timeline, параллакс, stagger | hero, ключевые секции, count-up цены | высокий для hero |
| **@formkit/auto-animate** | авто-плавность списков | корзина, слои кастомайзера, каталог-фильтры | средний, очень дёшево |
| **Embla Carousel (vue)** | плавные карусели | лента примеров, дропы, галерея товара | средний |

### 3.2 3D / вау-момент (после MVP-визуала)
| Three.js + **TresJS** (Vue) | вращающийся 3D-мокап изделия в hero или на странице товара | hero v2, премиум-фича | низкий приоритет, высокий вау |

### 3.3 UI и утилиты
- **Nuxt UI v4** — каркас компонентов (модалки, дропдауны, формы, табы). Обязательно кастомизировать под токены, не оставлять дефолт.
- **@nuxtjs/google-fonts** или локальные шрифты (Fontshare для Clash Display).
- **@nuxt/image** — оптимизация изображений (WebP, lazy, responsive) — критично для скорости с большими фото.
- **vue-sonner** — красивые toast-уведомления вместо алертов.
- **@vueuse/core** — утилиты (useScroll, useElementVisibility, useMagicKeys и т.д.).

### 3.4 Производительность
- `prefers-reduced-motion` — уважать системную настройку, отключать анимации для тех, кто их выключил (доступность + не укачивает).
- Lazy-load всех изображений ниже первого экрана.
- GSAP/Three подключать динамически (client-only, defer), чтобы не блокировать первую отрисовку.

## 4. Глобальные паттерны (сквозные компоненты)

### 4.1 Шапка (header)
- Прозрачная поверх hero (тёмного), при скролле — затемняется/появляется фон со стеклом (`--ink-glass` + backdrop-blur), уменьшается высота. Плавный переход 320ms.
- Лого слева (граффити-леттеринг — единственное место). Навигация по центру/справа: Каталог, Как это работает, [позже: Дизайнерам]. Справа — иконки: корзина (с бейджем количества), личный кабинет.
- Мобайл: бургер → полноэкранное меню с крупными пунктами, появление stagger.
- Магнитный hover на пунктах навигации (лёгкое притяжение к курсору).

### 4.2 Кнопки
- **Primary (CTA):** заливка `--ink-burgundy`, текст кремовый, `--shadow-burgundy`. Hover: чуть темнее + лёгкий подъём (translateY -2px) + усиление тени, 180ms. Активное нажатие: scale 0.98.
- **Secondary:** обводка 1.5px `--ink-black`, прозрачный фон. Hover: заливка чёрным, текст кремовый (инверсия).
- **Ghost:** только текст + подчёркивание, появляющееся по hover слева направо.
- Все кнопки — радиус `--radius-full` (пилюля) или `--radius-md` (выбрать одно и держать везде). Рекомендую пилюлю для CTA, md для второстепенных.
- Магнитный эффект на главных CTA (GSAP): кнопка слегка тянется к курсору в радиусе ~40px.

### 4.3 Карточки
- Радиус `--radius-lg`, фон белый/кремовый, `--shadow-sm` в покое.
- Hover: подъём translateY -8px, тень → `--shadow-lg`, изображение внутри scale 1.04 (overflow hidden), 320ms ease-out.
- Появление в сетке — каскадом (stagger 60–80ms между карточками).

### 4.4 Состояния (обязательны, их отсутствие = ощущение недоделанности)
- **Загрузка:** скелетоны (мерцающие плейсхолдеры формы контента), НЕ спиннер по центру пустой страницы.
- **Пусто:** иллюстрация/иконка + дружелюбный текст + CTA (см. копирайтинг пустых состояний в разделе 7).
- **Ошибка:** спокойный текст + кнопка повтора, без технических деталей.
- **Успех:** toast + микроанимация (галочка, лёгкий «отскок»).

### 4.5 Переходы между страницами
Nuxt page transitions: fade + лёгкий slide-up (16px), 320ms. Навигация ощущается как единое приложение, не перезагрузка.

### 4.6 Курсор и детали (опционально, усиливают премиум)
- Кастомный курсор на десктопе: маленькая точка, увеличивается над интерактивным (по желанию, не обязательно).
- Зерно/grain overlay на тёмных секциях (très лёгкое, 3–5% opacity) — добавляет фактуру, уводит от «плоского цифрового».
- Выделение текста (::selection) — бордо-фон, кремовый текст (мелочь, но бренд-деталь).

## 5. Лендинг — постраничный разбор по секциям

Весь копирайтинг ниже — финальный. Тон: уверенный, без канцелярита, на «ты», короткими ударными фразами. Аудитория 18–35 KZ.

### 5.1 Hero (первый экран)
**Визуал:** тёмный фон (`--ink-black` или глубокий бордо с лёгким градиентом/grain). Слева — крупный заголовок и CTA, справа — большое изображение/видео-петля реальной вещи с принтом (или анимированный мокап). На мобайле — изображение сверху, текст под ним. Лёгкий параллакс изображения при скролле.

**Анимация входа (GSAP timeline при загрузке):** лейбл fade-in → заголовок появляется по строкам со сдвигом вверх (stagger 80ms) → подзаголовок fade → кнопки появляются с лёгким overshoot → изображение проявляется с масштабом 1.06→1.0. Общая длительность ~1.2с.

**Копирайтинг:**
- Лейбл (mono, uppercase): `MERCH STUDIO · ALMATY · EST. 2025`
- H1: `ТВОЙ ПРИНТ. ТВОЯ ВЕЩЬ.`
- Подзаголовок: `Собери в браузере, увидь цену сразу, получи через пару дней. Печатаем от одной штуки — без партий и переплат.`
- CTA primary: `Создать свой принт`
- CTA secondary: `Смотреть каталог`
- Под кнопками (мелко, mono): `Доставка по Казахстану · Оплата онлайн · Тираж от 1`

### 5.2 Доверие-полоса (сразу под hero)
**Визуал:** тонкая горизонтальная полоса с 3–4 короткими буллетами через разделители-точки, mono-шрифт, появление при скролле. Снимает первичные возражения мгновенно.
**Копирайт:** `Качество, что не стыдно носить` · `Цвет согласуем перед печатью` · `Трек-номер в кабинете` · `Оплата только за подтверждённый заказ`

### 5.3 Лента примеров «Что создают на INKMADE»
**Визуал:** горизонтальная карусель (Embla) реальных работ — вещь крупным планом, lifestyle-фото. Бесконечная плавная прокрутка (marquee) или драг. Каждая карточка — фото + мелкая подпись (изделие + метод). Hover: пауза + лёгкий zoom.
**Заголовок секции:** лейбл `ГАЛЕРЕЯ` + H2 `Что создают на INKMADE`
**Подзаголовок:** `Реальные вещи реальных людей. Собери свою — добавим сюда.`

### 5.4 Каталог-превью «Выбери основу»
**Визуал:** сетка крупных карточек категорий/изделий (футболка, оверсайз, худи, свитшот, поло, кепка). Большое фото вещи, название, «от N ₸». Hover: подъём + zoom фото + появление кнопки `Кастомизировать`. Каскадное появление при скролле.
**Заголовок:** лейбл `КАТАЛОГ` + H2 `Выбери основу`
**Подзаголовок:** `Начни с изделия — принт добавишь в конструкторе.`
**Кнопка под сеткой:** `Весь каталог →`

### 5.5 Методы нанесения (образовательно-доверительный блок)
**Визуал:** 3 карточки-аккордеона или плитки с иконкой, названием, коротким описанием. При раскрытии/hover — пример применения. Появление по очереди.
**Заголовок:** лейбл `ТЕХНОЛОГИИ` + H2 `Как мы печатаем`
**Карточки (копирайт):**
- `DTG — прямая цифровая`: `Краска проникает в ткань. Полноцвет, фотопринт, мягко на ощупь. Идеально для хлопка.`
- `DTF — термоперенос`: `Яркий насыщенный принт на любой ткани. Держит цвет и стирку.`
- `Сублимация — full-print`: `Принт по всей поверхности без границ. Только для светлой синтетики.`

### 5.6 «Как это работает» (4 шага)
**Визуал:** горизонтальная (десктоп) / вертикальная (мобайл) линия из 4 шагов с крупными номерами, иконкой, заголовком, текстом. Соединительная линия «прорисовывается» при скролле (GSAP ScrollTrigger draw). Номера — display-шрифт, крупные.
**Заголовок:** лейбл `ПРОСТО` + H2 `От идеи до вещи — 4 шага`
**Шаги (копирайт):**
1. `Выбери изделие` — `Футболка, худи, свитшот, кепка. Выбери цвет и размер.`
2. `Добавь принт` — `Загрузи свой файл, выбери из каталога или набери текст.`
3. `Размести и оцени` — `Двигай, масштабируй, крути. Цена считается на лету.`
4. `Оплати и получи` — `Онлайн-оплата, печать, доставка по Казахстану.`

**Визуал:** 4 карточки с иконкой, заголовком, текстом. Сетка 2×2 (десктоп) / стек (мобайл). Появление каскадом.
**Заголовок:** лейбл `ПОЧЕМУ МЫ` + H2 `Спокойно за каждый заказ`

## 6. Остальные страницы — разбор

### 6.1 Каталог (`/catalog`, `/catalog/[category]`)
**Визуал:** крупная сетка карточек товаров (3–4 в ряд десктоп, 2 мобайл), много воздуха. Слева/сверху — фильтры (категория, цвет, метод, цена). Сортировка. Фильтры на мобайле — выезжающая панель снизу (bottom sheet).
**UX:** мгновенная фильтрация без перезагрузки (auto-animate перестроения сетки), сохранение фильтров в URL. Скелетоны при загрузке.
**Карточка товара:** фото на кремовом/нейтральном фоне, название, «от N ₸», бейдж метода/«новинка». Hover: смена фото на второй ракурс (front→back), подъём, кнопка `Кастомизировать`.
**Заголовок страницы:** H1 `Каталог` + подзаголовок-счётчик `N изделий · печать от одной штуки`.
**Пустой результат фильтра:** `Под такие фильтры ничего нет. Попробуй сбросить часть.` + кнопка `Сбросить фильтры`.

### 6.2 Страница товара (`/product/[id]`)
**Визуал:** двухколоночная. Слева — крупная галерея (основное фото + миниатюры/карусель, zoom по hover). Справа — название, цена, выбор цвета (свотчи с HEX), выбор размера (с ссылкой на размерную таблицу), краткое описание, методы для этого изделия, большой CTA.
**UX:** выбор цвета меняет фото галереи; недоступные размеры (stock 0) — приглушены. Размерная таблица — в модалке с реальными мерками (раздел размеров из схемы). «Добавить в избранное» (сердце).
**Главный CTA:** `Перейти в конструктор` (ведёт в кастомайзер с выбранными цветом/размером).
**Вторичный путь:** `Выбрать готовый принт` (для тех, кто не хочет свой файл — ведёт в каталог дизайнов с этим изделием).
**Блоки ниже:** описание (ткань, плотность, уход), отзывы (позже), «похожие изделия» (карусель).
**Копирайт-описание (пример, футболка):** `Плотный хлопок 100%, 190 г/м². Держит форму, не просвечивает, мягкий на ощупь. Печатаем твой принт методом DTG — полноцвет, без границ по фактуре.`

### 6.3 Кастомайзер (`/customize/[id]`) — ключевой экран, детально

**Раскладка (десктоп):** три зоны. Центр — крупный холст с мокапом изделия (нейтральный фон, тени, фактура). Слева — панель инструментов (выбор зоны, загрузка, библиотека принтов, текст). Справа — панель свойств (цвет изделия, размер, выбранный элемент, слои) + липкий блок цены и CTA внизу.
**Мобайл:** холст сверху, инструменты — нижняя панель с табами (Принт / Текст / Цвет / Размер), цена и CTA — липкая нижняя плашка.

**Инструменты и UX:**
- **Выбор зоны** (грудь/спина/рукав) — табы или кликабельные зоны на мокапе с подсветкой. Переключение — плавная смена ракурса мокапа (crossfade).
- **Загрузка принта** — drag-and-drop зона + кнопка. При загрузке: проверка DPI, тихий индикатор. Если мелко — мягкое предупреждение (не блок-ошибка, а подсказка): `Файл небольшой — на размере XL принт может выйти нечётким. Лучше загрузить покрупнее.`
- **Библиотека принтов** — сетка готовых дизайнов (бренд + дизайнеры), поиск/теги, превью на изделии по клику.
- **Текстовый инструмент** — ввод, выбор шрифта (включая дисплейный бренд-шрифт), цвет, размер. Живой предпросмотр на холсте.
- **Манипуляции** (Konva): свободно двигать, масштабировать за углы, вращать. Привязка к границам зоны (нельзя выйти). Лёгкие гайд-линии при выравнивании по центру.
- **Цвет изделия** — свотчи, смена цвета мокапа в реальном времени (crossfade), принт остаётся.
- **Слои** — список элементов (если несколько принтов/текст), порядок, удаление. auto-animate.

**Цена (важно для конверсии):** липкий блок, число меняется с count-up анимацией при каждом изменении. Под ценой — расшифровка (изделие + нанесение + зоны). CTA `Добавить в корзину`.

**Микро-детали:** при добавлении в корзину — превью «улетает» в иконку корзины (анимация), toast `Добавили в корзину`. Кнопка `Поделиться` — генерит ссылку на дизайн (копирайт toast: `Ссылка скопирована — покажи друзьям`).

**Пустой холст (онбординг):** подсказка по центру `Загрузи принт, выбери из каталога или набери текст` + три быстрые кнопки.

### 6.4 Корзина (`/cart`)
**Визуал:** список позиций с превью композиции (скриншот дизайна на изделии), название, параметры (цвет/размер/метод/зоны), цена, количество (степпер), удаление. Справа/снизу — итог, промокод, CTA `Оформить заказ`.
**UX:** изменение количества/удаление — auto-animate, пересчёт итога с count-up. Пустая корзина: `Пока пусто. Самое время собрать что-нибудь своё.` + CTA `В каталог`.

### 6.5 Checkout (`/checkout`)
**Визуал:** чистая одноколоночная форма с шагами (контакты → доставка → оплата) или аккордеон-степпер. Справа — сводка заказа (липкая). Минимум полей, максимум ясности.
**UX:** гость доходит до этого шага, логин/регистрация — здесь (мягко, «войди или продолжи как гость с быстрой регистрацией»). Чекбокс согласия (ToS + индивидуальность изделия) — обязателен, без него CTA неактивен. Дисклеймер о цвете — рядом с оплатой.
**Копирайт согласия:** `Я понимаю, что вещь изготавливается по моему дизайну и не подлежит возврату как товар надлежащего качества. Претензии по качеству и браку принимаются.`
**CTA:** `Оплатить N ₸`.

### 6.6 Страница заказа / трекинг (`/order/[id]`)
**Визуал:** крупный статус-трекер (горизонтальная линия этапов: Оформлен → В производстве → Упаковка → В пути → Доставлен) с анимированным прогрессом. Ниже — детали заказа, превью, трек-номер, чек.
**UX:** текущий этап подсвечен бордо, пройденные — отмечены, будущие — приглушены. Анимация заполнения линии.
**Копирайт статусов (для клиента, укрупнённо):** `Оформлен` · `Печатаем` · `Упаковываем` · `В пути` · `Доставлен`.

## 7. Копирайтинг: тон голоса и микрокопи

### 7.1 Тон голоса
- На «ты», уверенно, дружелюбно, без канцелярита и пафоса.
- Короткие ударные фразы. Глаголы действия (собери, носи, создай, получи).
- Продаём результат и эмоцию («носи своё»), не процесс («сервис онлайн-печати»).
- Честно про ограничения (цвет, возврат) — это повышает доверие, а не снижает.
- Без англицизмов-жаргона там, где есть русское слово. «Конструктор», не «кастомайзер» (в интерфейсе для клиента).

### 7.2 Микрокопи (кнопки, подсказки, состояния)
- Кнопки: глагол + объект. `Создать свой принт`, `Добавить в корзину`, `Оформить заказ`, `Перейти в конструктор`.
- Загрузка: `Секунду, собираем…` (не «Loading»).
- Успех корзины: `Добавили. Собрать ещё или оформить?`
- Шаринг: `Ссылка скопирована — покажи друзьям`.
- Ошибка оплаты: `Оплата не прошла. Попробуй ещё раз или другой способ.`
- Пустые состояния — см. по страницам выше, всегда: дружелюбная фраза + выход (CTA).

### 7.3 Принцип заголовков секций
Каждая секция: mono-лейбл (одно слово, uppercase) + крупный H2 (выгода/действие, не «название блока»). Пример: не «Преимущества», а лейбл `ПОЧЕМУ МЫ` + H2 `Спокойно за каждый заказ`.

## 8. Анимации — сводная спецификация

| Элемент | Триггер | Эффект | Длительность / easing |
|---|---|---|---|
| Hero | загрузка | timeline: лейбл→заголовок по строкам (stagger 80ms)→подзаг→кнопки(overshoot)→фото(scale 1.06→1) | ~1.2с, ease-out |
| Секции лендинга | вход в вьюпорт | fade + slide-up 24px | reveal 700ms, ease-out, once |
| Карточки сетки | вход | каскад stagger 60–80ms | base 320ms |
| Карточка hover | hover | подъём -8px, тень lg, фото scale 1.04 | base, ease-out |
| Кнопка CTA | hover | подъём -2px, тень+, (магнит GSAP в радиусе 40px) | fast |
| Кнопка | click | scale 0.98 | fast |
| Линия «как работает» | скролл | draw линии по ScrollTrigger | по скроллу |
| Цена в кастомайзере | изменение | count-up числа | base |
| Смена цвета изделия | выбор | crossfade мокапа | base |
| Добавление в корзину | click | превью «улетает» в иконку + bounce бейджа | slow |
| Список (корзина/слои) | add/remove | auto-animate | base |
| Скролл страницы | всегда | Lenis инерция | — |
| Переход страниц | навигация | fade + slide-up 16px | base |
| Статус-трекер заказа | загрузка | заполнение линии до текущего этапа | slow |
| Аккордеон FAQ | клик | grid-rows раскрытие + поворот иконки | base |

Все анимации отключаются при `prefers-reduced-motion: reduce`.

## 9. Интересные фишки и решения (усиливают продукт)

Сверх базового — то, что выделит INKMADE и поднимет вовлечение/конверсию.

1. **Живой count-up цены** в кастомайзере — число «накручивается» при каждом изменении. Делает ценообразование осязаемым и честным.
2. **«Улетающее в корзину» превью** — микроанимация добавления, дарит ощущение отклика.
3. **Шаринг дизайна ссылкой** прямо из конструктора с авто-скриншотом композиции — встроенный вирусный канал. Открывший ссылку видит готовый дизайн и может заказать или изменить.
4. **Магнитные CTA** — главные кнопки тянутся к курсору. Дорогая деталь, запоминается.
5. **Hero-видео процесса печати** — петля, как принт ложится на ткань. Доказывает собственное производство и залипательно.
6. **Размерный помощник** — на странице товара мини-калькулятор «введи рост/обхват → подскажем размер» (из fit_* полей размерной сетки).
7. **Подсказки качества в реальном времени** — конструктор мягко ведёт: «принт мелковат», «сдвинь ниже центра — смотрится лучше». Помощник, не цензор.
8. **Сезонные дропы на главной** — баннер-таймер до выхода коллекции (FOMO), управляется из админки.
9. **Подарочный режим** — переключатель «это подарок» в checkout: упаковка, открытка с текстом, доставка на адрес получателя, скрытие цены в посылке.
10. **Скелетоны вместо спиннеров** — контент проявляется формой, ощущение скорости.
11. **Grain/зерно на тёмных секциях** — лёгкая фактура, уводит от плоского цифрового вида.
12. **Прогресс-индикатор сборки** в кастомайзере — тонкая полоса «изделие → принт → размещение → готово», подсказывает следующий шаг новичку.
13. **«Недавно создано»** — лента последних публичных дизайнов на главной (соц-доказательство, что платформа живая).
14. **Тёмная/светлая подача по секциям** — чередование тёмных (бренд, эмоция) и светлых (каталог, функция) секций создаёт ритм и не утомляет.

## 10. Адаптив и мобайл (приоритет — мобайл)

Аудитория 18–35 KZ заходит преимущественно с телефона. Мобильная версия — не «ужатый десктоп», а первичный сценарий.
- Брейкпоинты: mobile ≤640, tablet ≤1024, desktop >1024.
- Кастомайзер на мобайле: холст сверху, инструменты — нижняя таб-панель, цена+CTA — липкая нижняя плашка. Жесты: pinch-to-zoom, drag принта пальцем.
- Фильтры каталога — bottom sheet.
- Меню — полноэкранное со stagger.
- Тач-цели ≥44px. Hover-эффекты заменяются на active/tap-состояния.
- Изображения — responsive через @nuxt/image, WebP, нужные размеры под вьюпорт.

## 11. Доступность и качество

- Контраст текста ≥ WCAG AA (проверить бордо на кремовом и наоборот).
- Фокус-состояния на всех интерактивных (видимая обводка).
- Альты на изображениях, лейблы на полях.
- `prefers-reduced-motion` уважается.
- Семантический HTML, нав по клавиатуре.
- Скорость: LCP < 2.5с (оптимизация hero-изображения критична), отложенная загрузка тяжёлых либ (GSAP/Three) client-only.

## 12. Порядок внедрения редизайна (для разработчика)

**Этап 1 — фундамент (быстро, бесплатно, большой эффект):**
Внедрить токены (раздел 2), заменить дисплейный шрифт, увеличить типографику/воздух/отступы секций, настроить ::selection, page transitions.

**Этап 2 — движение (максимальный эффект/усилие):**
Lenis (плавный скролл) → @vueuse/motion (появление секций при скролле) → hover-состояния карточек и кнопок → auto-animate на списках.

**Этап 3 — hero и ключевые секции:**
Качественное hero-изображение/видео → GSAP timeline входа → магнитные CTA → draw-линия в «как работает» → count-up цены в кастомайзере.

**Этап 4 — полировка:**
Скелетоны, пустые/ошибочные состояния, toast (vue-sonner), «улетающее в корзину» превью, grain на тёмных секциях, статус-трекер заказа.

**Этап 5 — вау (после доведения базы):**
3D-мокап (TresJS) в hero/товаре, кастомный курсор, hero-видео процесса.

**Главный принцип:** сначала весь сайт получает фундамент и движение (этапы 1–2) — это уже превращает «шаблон» в «продукт». Вау-эффекты (этап 5) — только после того, как база безупречна. Тяжёлый медленный 3D хуже лёгкого быстрого 2D.

## 13. Резюме для разработчика

Направление: премиальный стрит-минимализм — дорогой чистый каркас + бренд-акценты. Три закона: воздух/крупность, один акцент на экран, движение = качество.

Ключевые библиотеки в порядке внедрения: Lenis → @vueuse/motion → GSAP → auto-animate → Embla → @nuxt/image → vue-sonner. 3D (TresJS) — потом.

Шрифт: заменить граффити на характерный гротеск (Clash Display/General Sans), граффити оставить в логотипе.

Весь копирайтинг в документе — финальный, вставлять как есть. Тон: на «ты», результат вместо процесса, честно про ограничения.

Мобайл — первичный сценарий, не ужатый десктоп.

Сначала фундамент и движение по всему сайту, вау-эффекты — после.
